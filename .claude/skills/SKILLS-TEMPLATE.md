# Skills 模板指南

## 1. 验证 Skill 模板（test-*.md）

用于测试和验证特定功能

```markdown
# 测试功能名称

## 描述
说明这个 skill 的用途

## 使用
/test-xxx

## 验证步骤
1. 步骤 1
2. 步骤 2
3. 步骤 3

## 预期结果
✅ 预期的结果 1
✅ 预期的结果 2

## 关键文件
- src/components/Xxx.jsx
- src/hooks/useXxx.js
```

## 2. 修改 Skill 模板（modify-*.md）

用于指导如何修改特定功能

```markdown
# 修改功能名称

## 描述
需要改什么，为什么改

## 修改范围
- 组件: Xxx.jsx
- 样式: colors/spacing
- 逻辑: 处理方式

## 修改步骤
1. 修改 A 地方，改成 B 效果
2. 修改 C 地方，加上 D 功能
3. 测试效果

## 参考资料
相关的代码位置和注意事项
```

## 3. 功能 Skill 模板（feature-*.md）

用于实现新功能

```markdown
# 新功能名称

## 功能描述
用户能做什么，有什么好处

## 实现范围
- 新增文件: xxx
- 修改文件: yyy
- 依赖: zzz

## 实现检查清单
- [ ] 核心逻辑完成
- [ ] UI 样式完成
- [ ] 功能测试通过
- [ ] 代码提交推送

## 使用示例
```

## 4. 修复 Skill 模板（fix-*.md）

用于修复已知问题

```markdown
# 修复问题描述

## 问题
发生了什么，在哪个组件/功能

## 根本原因
为什么会这样

## 解决方案
怎么修

## 修复检查清单
- [ ] 原问题消失
- [ ] 没有引入新问题
- [ ] 相关测试通过

## 验证方式
怎么确认问题已修复
```

---

## 使用方式

1. **创建 skill 文件**
   在 `.claude/skills/` 目录下创建 `名称.md` 文件

2. **调用 skill**
   在对话中使用 `/skill-name` 命令

3. **Claude 会识别**
   根据 skill 中的指导来结构化地完成任务

## 推荐命名约定

- `test-xxx.md` - 测试相关
- `modify-xxx.md` - 修改相关
- `feature-xxx.md` - 新功能
- `fix-xxx.md` - 修复问题
- `setup-xxx.md` - 初始化/配置
